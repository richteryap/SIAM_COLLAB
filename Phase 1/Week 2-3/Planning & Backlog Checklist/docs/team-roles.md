## Role Descriptions

### PM / Scrum Master
- Runs daily standups and sprint planning/review meetings
- Tracks progress on the sprint board
- Removes blockers for the team
- Maintains backlog and schedule

### QA Lead
- Writes and maintains test cases
- Reviews code for quality and edge cases
- Manages the defect log
- Ensures tests pass before merging

### DevOps Lead
- Manages CI/CD pipeline (GitHub Actions)
- Handles deployment and environment setup
- Monitors build status and fixes pipeline issues
- Manages release tagging and versioning

### Docs Lead
- Maintains all `/docs` files (backlog, plans, notes)
- Writes release notes and changelogs
- Ensures README is up to date
- Reviews PR descriptions for completeness

### Lead Developer
- Leads code architecture decisions
- Reviews all PRs for code quality
- Helps teammates debug issues
- Ensures coding standards are followed
