# Sprint 1 Plan — Flashcard Quiz App

## Sprint Goal
Deliver a working flashcard app where users can create, view, flip, delete, and edit flashcards.

## Sprint Duration
2 weeks

## Selected Stories (5 stories, 13 points)

| Story ID | Title               | Points | Owner          | Status      |
|----------|---------------------|--------|----------------|-------------|
| US-01    | Create a Flashcard   | 3      | [Member 1]     | To Do       |
| US-02    | View All Flashcards  | 2      | [Member 1]     | To Do       |
| US-03    | Flip a Flashcard     | 3      | [Member 2]     | To Do       |
| US-04    | Delete a Flashcard   | 2      | [Member 3]     | To Do       |
| US-05    | Edit a Flashcard     | 3      | [Member 3]     | To Do       |

**Total Sprint Points:** 13

## Rationale
Sprint 1 covers all the core CRUD operations (Create, Read, Update, Delete) plus the card flip interaction. These are the building blocks — the quiz feature (Sprint 2) depends on having a working deck of cards first.

## Definition of Done
- Feature code is written and committed
- At least 1 unit test per story passes
- Code reviewed via Pull Request
- No console errors in the browser
- Feature demo'd to team

## Daily Standup Schedule
- **When:** Monday, Wednesday, Friday — 10 minutes
- **Format:** What I did → What I'll do → Blockers
