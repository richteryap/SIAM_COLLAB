## Rationale
Sprint 1 covers all the core CRUD operations (Create, Read, Update, Delete) plus the card flip interaction. These are the building blocks — the quiz feature (Sprint 2) depends on having a working deck of cards first.

## Definition of Done
- Feature code is written and committed
- At least 1 unit test per story passes
- Code reviewed via Pull Request
- No console errors in the browser
- Feature demo'd to team
